> 原文：https://gist.github.com/karpathy/442a6bf555914893e9891c11519de94f
> 这是一个创意文件，设计成可以复制粘贴到你自己的 LLM Agent 中。

## 核心理念

大多数人使用 LLM 和文档的体验就像 RAG：你上传一组文件，LLM 在查询时检索相关区块，生成答案。这确实有效，但 LLM 是在每个问题上从零重新发现知识。没有积聚。提出一个需要综合五份文档的微妙问题，LLM 每次都必须找到并拼凑出相关碎片。没有任何积累。NotebookLM、ChatGPT 文件上传以及大多数 RAG 系统都是这样工作的。

这里的理念不同。LLM 不仅仅是在查询时从原始文档中检索，而是逐步构建和维护一个持久维基——一个结构化、相互关联的 markdown 文件集合，位于你和原始源之间。

当你添加一个新源时，LLM 不仅仅是索引它以便以后检索。它阅读这些数据，提取关键信息，并将其整合进现有维基——更新实体页面，修订主题摘要，指出新数据与旧主张相矛盾的地方，强化或挑战不断演变的综合。这些知识会被编译一次，然后保持最新，而不是每次查询都重新推导。

关键区别在于：维基是一个持续存在的、不断累积的伪造品。交叉引用已经存在了。矛盾已经被指出。综合内容已经反映了你读过的所有内容。你添加的每一个来源和每一个问题，维基内容都会越来越丰富。

你从不（或很少）自己写维基——LLM 负责编写和维护所有内容。你负责寻找、探索并提出正确的问题。LLM 负责所有繁重的工作——总结、交叉引用、归档和簿记，这些都让知识库随着时间变得真正有用。实际上，我一边开着 LLM Agent，另一边开着 Obsidian。LLM 会根据我们的对话进行编辑，我则实时浏览结果——点击链接、查看图表视图、阅读更新后的页面。**Obsidian 是 IDE，LLM 是程序员，维基就是代码库。**

## 适用场景

- **个人**：追踪自己的目标、健康、心理、自我提升——整理日记、文章、播客笔记
- **研究**：深入探讨一个主题——阅读论文、文章、报告，构建综合维基
- **读书**：边读边整理章节，构建角色、主题、情节线索页面
- **业务/团队**：LLM 维护的内部维基，由 Slack 线程、会议记录、项目文档提供信息
- **其他**：竞争分析、尽职调查、行程规划、课程笔记、兴趣深入

## 架构：三层

**原始资料** —— 你精心整理的原始文件收藏。文章、论文、图片、数据文件。这些是不可变的——LLM 从中读取，但从不修改。这就是你的真相来源。

**维基** —— 由 LLM 生成的 markdown 文件目录。摘要、实体页、概念页、比较、概述、综合。LLM 完全拥有这一层。它创建页面，当新资料来源到来时更新，维护交叉引用，并保持一切一致性。你读；LLM 写。

**模式** —— 一份文档（CLAUDE.md 或 AGENTS.md），告诉 LLM 维基的结构、惯例以及工作流程。这是关键的配置文件——正是它让 LLM 成为一个有纪律的维基维护者，而不是普通的聊天机器人。你和 LLM 会随着时间推移共同进化。

## 运营

### Ingest（吞下）

你把一个新源放进原始集合，然后让 LLM 处理它。LLM 会阅读源代码，与你讨论关键要点，在维基中写摘要页，更新索引，更新维基中相关的实体和概念页面，并在日志中添加条目。一个来源可能覆盖 10 到 15 个维基页面。

### Query（查询）

你会对着维基提问。LLM 会搜索相关页面，阅读它们，并综合答案并引用。好的答案可以作为新页面重新归档到维基——你的探索也能积累知识库。

### Lint（绒毛）

定期让 LLM 对维基进行健康检查：页面间矛盾、陈旧说法、孤立页面、缺失交叉引用、可填补的数据空白。

## 索引与记录

- **index.md** —— 以内容为导向。维基中所有内容的目录，按类别组织。LLM 每次导入时更新。查询时先读索引找到相关页面。
- **log.md** —— 按时间顺序。只追加的时间和记录：发生了什么操作。可以轻松用 Unix 工具解析。

## 技巧与窍门

- Obsidian Web Clipper：将网页转为 markdown 的浏览器扩展
- 本地下载图片：设置附件目录到 `raw/assets/`
- Obsidian 图视图：查看维基结构的最佳方式
- Marp：基于 Markdown 的幻灯片格式
- Dataview：通过 frontmatter 运行查询
- 维基本质是 git 仓库——免费版本历史

## 为什么有效

维护知识库最繁琐的部分不是阅读或思考，而是**记账**。更新交叉引用、保持摘要最新、指出矛盾、保持数十页一致性。人类放弃维基是因为维护负担增长得比价值快。LLM 不会感到无聊，不会忘了更新交叉引用，一次可以处理 15 个文件。

人类的工作是整理资料，引导分析，提出好问题，思考这一切的意义。LLM 的工作就是其他所有事情。

## 注释

本文档为有意抽象的。它描述的是想法，而不是具体的实现方式。具体的目录结构、模式惯例、页面格式、工具——这些都取决于你的领域、偏好以及你选择的 LLM。正确的使用方式是与你的 LLM Agent 共享，并共同协作实现一个符合你需求的版本。
